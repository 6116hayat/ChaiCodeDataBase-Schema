# 🗄️ Database Designs

This repository contains a collection of real-world database design projects built as part of a structured 5-day challenge from Chai Code.

Each project focuses on modeling practical systems and understanding how data is structured, related, and managed in real applications.

## 📁 Projects Overview

### 📦 Instagram Store

A database design for an Instagram-based store handling:

- Thrifted products (unique items)
- Handmade products (multiple stock)
- Orders, customers, and payments

---

### 💪 Fitness Influencer Platform

A system designed for fitness coaches and clients, including:

- Trainer–client relationships
- Workout and diet plans
- Progress tracking and subscriptions

---

### 🏥 Clinic Management System

A system designed for managing clinic operations, including:

- Doctor–patient interactions
- Appointment booking and consultations
- Diagnostic tests and report management
- Billing, invoices, and payments

---

### 🅿️ Comic-Con Parking Management System

A system designed for managing large-scale event parking, including:

- Vehicle entry and exit tracking
- Multi-zone and multi-level parking management
- Smart parking spot allocation based on vehicle type
- Reserved parking for VIPs, staff, exhibitors, and cosplayers
- Parking sessions with ticket generation
- Pricing calculation and payment handling

---

### 🛗 Smart Elevator Control System

A system designed for managing intelligent elevator operations in large buildings, including:

- Multi-building and multi-elevator management
- Floor-level request generation and tracking
- Smart ride allocation to the most optimal elevator
- Elevator-to-floor mapping for zoned and express lifts
- Ride lifecycle tracking (request → assignment → trip)
- Real-time elevator status monitoring (idle, moving, maintenance, offline)
- Maintenance tracking with historical records
- Ride logs for performance analytics and system optimization

---

## 🧠 What This Repo Covers

- Entity-Relationship (ER) Modeling
- Table structures and schema design
- Relationships (1:1, 1:N, N:M)
- Basic normalization
- Real-world data constraints
